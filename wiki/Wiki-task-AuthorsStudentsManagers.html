Component Purpose

The AuthorSection, ManagerSection, StudentSection components are a user interface element for displaying and managing a list of authors or managers or authors. The interface provides two different views for the user: a list of unique persons and a list of all persons in this category.

Key Functionality

  • Data Loading

    When the component first loads, it fetches two sets of data simultaneously: a list of unique IDs and a list of all IDs, which also include author IDs. It then fetches the full profile data (name, email, etc.) for each unique author ID.

  • View Switching

    The user can switch between the "Unique" view and the "All" view using a radio button group. The view change updates the list displayed below.

  • Display Logic

    • Unique Person View: This view shows a list of each distinct person's name and email. If the same person appears on the 'All' list more than once the duplications do not appear on the 'Unique' List. The total count of unique persons in each category (authors, managers, students) is displayed in the title.

    • All Authored Tasks View(available in the Author component & the Task component): This view shows a list of every authored task. For each task, it displays the corresponding author's name and email. This means an author's information can appear multiple times if they have authored more than one task. The total count of all authored tasks is shown in the title.

  • User Feedback

    The component displays a "Loading authors/students/managers..." message while it is fetching data from the backend. If no authors are found in either view, it displays a message indicating that there is no data to show for the selected view.
