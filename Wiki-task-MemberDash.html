Component Purpose

The MemberDashboard component serves as the primary landing page for a logged-in user. It provides a comprehensive overview of the user's profile, key statistics related to their progress, a list of their active tasks, and a collection of quick actions.

Key Functionality

  • User Authentication and Profile Display

    It uses an authentication context to retrieve and display the user's profile details, including their name, email, avatar, join date, and last login time. This information is presented in a dedicated section on the left side of the dashboard. It also includes an "Edit Profile" button.

  • Dynamic Statistics and Data Fetching

    The component fetches real-time data from a database (Supabase) to show the user's active and available tasks. It presents these stats in a series of cards alongside mock data for completed tasks and learning hours, providing a quick, visual summary of their performance.

  • Task Management Overview

    The dashboard displays a list of the user's assigned tasks (currently using mock data). Each task entry shows its name, a description, and a progress bar indicating the current stage. A sub-section for each task details the name and description of the current stage, with buttons to navigate between stages.

  • Quick Actions and Navigation

    A "Quick Actions" section provides a set of buttons for common user actions, such as inviting a friend, copying a recruitment link, checking rewards, and contacting support. This section streamlines the user experience by offering shortcuts to important features.
