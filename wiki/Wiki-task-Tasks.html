Component Purpose

The TasksSection component is a user interface element designed to display and manage a list of all available tasks. It fetches task data from a backend and provides sorting functionality to help users organize and navigate the list.

Key Functionality

  • Data Loading and Refreshing

    When the component first loads, it concurrently fetches all tasks and assignments from a data service. It also retrieves the profile information for all unique authors associated with these tasks. A "Refresh" button is provided to allow the user to manually reload the data at any time.

  • Task Sorting

    The component provides two buttons for sorting the tasks alphabetically by name: "A-Z" for ascending order and "Z-A" for descending order. The currently active sort order is highlighted to provide visual feedback to the user.

  • Display of Task Information

    Each task is displayed within its own card, showing the task's name, a truncated version of its ID, a description (if available), and a link to an external URL (if provided). The card also includes the name of the author and the creation date of the task. If a task ID is missing or an author's name is not found, the component displays placeholder text like "---" or "Unknown".

  • User Feedback

    While the data is being fetched, the component displays a "Loading tasks..." message. If an error occurs during the data retrieval, it logs the error to the console. The "Refresh" button is disabled during the loading process to prevent multiple simultaneous requests.
