Noting notable notes

They said it was easy

Click in the input panel, type your note, click [save/send]. The note will be stored in the database.

If you click on any of the little boxes that name will be attached to the note and stored in the database. Later you will be able to search for notes with any of these tags

Tags - attached to the note

The main tags currently show that your note is a bug report (reporting something wrong with the system). The other main tags are me which means it is a note to yourself, else which means the note is directed towards someone else, reply which means it should be a reply to an existing note

Tags - importance

At the top right is a choice of setting how important you regard this note. The importance is usually set at 3. If your notes is less important than average click a lower number. If more important than average click a higher number. (If you really want o go to 6 you will have to wait until this is implemented)

Tags - events

This group of tags includes diary which is a very commonly used tag if you are keeping track of a project. It implies a commentary on actions taken and progress or lack of it. triumph,disaster, and rant are extreme tags

Tags - process, business, & resource

All tags are treated the same in the system and the name will, in future be editable so that you can define the tagging system that is good for your project or life

Tags - Pending, abandoned, completed

These are available after saving/sending the note. These can be applied after reading a note. To apply these tags is different because you are marking the message after considering it. To do this you need to find the saved messages by scrolling down. Then click on a message to show a pending symbol, which is a question mark.If you repeat the click the various possible symbols will cycle. Click once for green ? again for a completed tick symbol, again returns to a red ? and again brings a red x representing abandoning/ignoring the note

The Lab is a minor part of a larger project, but the whole project is built in pieces in the belief that any piece could turn out to have its own value

The lab is a place to keep your notes and also keep links to websites. The notes can be tagged and later found by searching for notes with specific tags. The Lab begins with a default set of tags, but you can edit them to suit your interests or projects. If you are collecting info about cars you may want tags that are specific to cars and you may want web links to car parts dealers or car showrooms or photos of cars. If you are writing about a war you may want to store quotes and have tags specifying certain periods or places. You can edit your own tag list. And once the system knows it will prevent you mispelling them. In that way you will be able to find the note because tags are standardised.

The default setting for the Lab is aimed at software developers. Why? Because we are the first users of the product and so that is how we have edited it.

How to include a new webpage:

  • Create your HTML file in the appropriate folder (usually /pages or /src)
  • Edit the navigation configuration file to include your new page
  • Add the route to your routing configuration
  • Commit and push to your GitHub repository
  • Lovable will automatically recognize and deploy your new page

Adding a new script:

  • Scripts should be placed in the /scripts or /src/utils folder
  • Ensure your script follows the required format (JavaScript/TypeScript)
  • Reference your script in the appropriate HTML or component file
  • Test locally before committing
  • Push to GitHub for automatic deployment
