But you aren't here to read the moans of an ai assisted software developer. So what is this Lab thing all about?

The Lab is a minor part of a larger project, but the whole project is built in pieces in the belief that any piece could turn out to have its own value

The lab is a place to keep your notes and also keep links to websites. The notes can be tagged and later found by searching for notes with specific tags. The Lab begins with a default set of tags, but you can edit them to suit your interests or projects. If you are collecting info about cars you may want tags that are specific to cars and you may want web links to car parts dealers or car showrooms or photos of cars. If you are writing about a war you may want to store quotes and have tags specifying certain periods or places. You can edit your own tag list. And once the system knows it will prevent you mispelling them. In that way you will be able to find the note because tags are standardised.

The default setting for the Lab is aimed at software developers. Why? Because we are the first users of the product and so that is how we have edited it.

How to include a new webpage:

  • Create your HTML file in the appropriate folder (usually /pages or /src)
  • Edit the navigation configuration file to include your new page
  • Add the route to your routing configuration
  • Commit and push to your GitHub repository
  • Lovable will automatically recognize and deploy your new page

Adding a new script:

  • Scripts should be placed in the /scripts or /src/utils folder
  • Ensure your script follows the required format (JavaScript/TypeScript)
  • Reference your script in the appropriate HTML or component file
  • Test locally before committing
  • Push to GitHub for automatic deployment
